Use Conventional Commits format. Examples:
- `feat: add user authentication module`
- `fix: resolve memory leak in connection handler`
- `docs: update API documentation`
- `refactor: extract payment logic to separate service`
- `test: add unit tests for user validator`

Format: `<type>(<scope>): <description>`
- `type`: feat, fix, docs, style, refactor, test, chore, perf, ci, build
- `scope`: optional, the component being changed
- `description`: short, imperative tense

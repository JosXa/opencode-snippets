# Release Command

Guide through the complete release workflow for `opencode-snippets`.

## Usage

`/release` - analyze commits, prepare the release, and publish via tag push

## Instructions

You are executing the release workflow. Follow the repo instructions in `AGENTS.md` over this file if they ever disagree.

### Pre-Release Checks

1. Verify CI is healthy:
   - Run `gh run list --limit 5`
   - If the latest relevant run failed, STOP and inform the user

2. Inspect the worktree:
   - Run `git status --short`
   - If there are unrelated changes, STOP and ask before touching them
   - If the release work itself is still uncommitted, continue and prepare the release

3. Determine the version bump from conventional commits since the last tag:
   - Run `git log <last-tag>..HEAD --oneline`
   - `BREAKING CHANGE` or `type!:` means major
   - `feat:` means minor
   - everything else means patch
   - If a major bump is detected, STOP and ask for confirmation

### Release Implementation

Execute these steps in order:

1. Prepare release changes:
   - Update `package.json`
   - Manually update schema URLs in:
     - `README.md`
     - `src/config.ts`
     - `skill/snippets/SKILL.md`
   - Update any release workflow docs affected by current repo rules

2. Validate before committing:
   - Run `bun run format:check`
   - If warnings are only in unmodified files, ask whether to fix them or proceed
   - Run the relevant test/build commands for the change set

3. Commit the functional/docs changes first using a conventional commit message

4. Commit the version bump separately:
   - Commit message: `chore: bump version to vX.Y.Z`

5. Push commits to `origin/main`

6. Create and push the tag:
   - `git tag vX.Y.Z`
   - `git push origin vX.Y.Z`

7. Monitor release automation:
   - The publish workflow and GitHub release notes are generated by CI on tag push
   - Do not ask the user for release notes
   - Use `gh run watch` for the release workflow if monitoring is requested

8. After release succeeds:
   - Update the user's global schema URL in `C:\Users\josch\.config\opencode\snippet\config.jsonc`

### Recovery: Tag Pushed While CI Failing

If a tag was pushed while CI is failing, delete it immediately:

```bash
git tag -d vX.Y.Z
git push origin :refs/tags/vX.Y.Z
```

Then fix the issue and restart from the push step.

### Important Notes

- Always use Bun commands, never npm
- `bun run format:check` is required before every commit
- Publishing to npm is automated via CI on tag push
- GitHub release notes are auto-generated from conventional commits
- Version format is `X.Y.Z` in `package.json` and `vX.Y.Z` for the git tag
- Repository: https://github.com/JosXa/opencode-snippets

### Error Handling

If any step fails:
1. Stop immediately
2. Inform the user what failed
3. Provide the exact recovery step
4. Do not continue automatically
